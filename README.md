Sonexa: Plataforma de Gestão de Coleções Fonográficas

  Sobre o Projeto
Este projeto foi desenvolvido no âmbito da unidade curricular de Projetos de Sistemas de Informação (2025/2026). O objetivo principal foi criar uma solução robusta
para colecionadores de música em formato físico (Vinil, CD e Cassete), permitindo-lhes não só catalogar os seus itens, mas também explorar metadados detalhados da indústria musical.
A aplicação funciona como um inventário digital inteligente, onde a integridade dos dados e a experiência do utilizador foram as nossas prioridades.

  A Equipa
O desenvolvimento foi fruto de um trabalho colaborativo entre eu e mais três colegas. Acreditamos que a comunicação e a divisão eficiente de tarefas foram
fundamentais para entregar uma plataforma funcional e coesa dentro dos prazos estipulados.

  Stack Tecnológica
Frontend: Angular (Interface reativa e componentes modulares).
Backend: Node.js & Express (API RESTful desenhada com boas práticas).
Base de Dados: MongoDB (Flexibilidade para gerir versões complexas de álbuns).
Controlo de Versões: Git (Utilizado de forma rigorosa para gestão de conflitos e organização de sprints).

  Funcionalidades Principais
Gestão de Coleção: Adição de álbuns específicos por suporte físico (CD, Vinil, Cassete) e versão (EAN-13).
Metadados Avançados: Integração lógica com identificadores padrão da indústria como MBID, ISRC e ISNI.
Sistema de Contas: Autenticação segura com cifragem de passwords e perfis personalizáveis.
Exploração e Descoberta: Motores de pesquisa para artistas e álbuns com sugestões parciais.
Listas Personalizadas: Criação de coleções temáticas (ex: "Favoritos") independentes do inventário físico.
Curadoria Colaborativa: Sistema de submissão de novas versões para validação, permitindo a atualização contínua da base de dados.

  Metodologia de Desenvolvimento
O projeto foi executado seguindo o modelo Scrum, dividido em 3 Sprints de duas semanas cada. Esta abordagem permitiu-nos:
Priorizar as User Stories críticas logo no início.
Garantir entregas incrementais e funcionais.
Manter um código limpo e escalável através de revisões e melhoramentos constantes.
